使用 **Nuxt**？请看 Nuxt 模式的安装说明。
::

`@movk/nuxt` 不仅是 Nuxt 模块，还提供 Vite 插件（`@movk/nuxt/vite`）与 Vue 插件（`@movk/nuxt/vue-plugin`），让独立 UI 组件、AutoForm / DataTable、主题与非服务端 Composables 在纯 Vue + Vite 项目中直接可用。

::warning
Vue / Vite 模式**不包含 API 集成域**（`useApiFetch` / `useLazyApiFetch` / `useClientApiFetch`、上传下载、`$api` 插件、鉴权与错误 toast 等），它们依赖 Nuxt 服务端运行时，仅在 Nuxt 模式下可用。详见下方[差异对照](#与-nuxt-模式的差异)。
::

## 安装

::steps{level="3"}

### 安装依赖

::code-group
```bash [pnpm]
pnpm add @movk/nuxt @nuxt/ui zod tailwindcss
```

```bash [npm]
npm install @movk/nuxt @nuxt/ui zod tailwindcss
```

```bash [yarn]
yarn add @movk/nuxt @nuxt/ui zod tailwindcss
```
::

vue-router 默认启用（`router` 选项控制）。若使用默认值，请同时安装 `vue-router`：

::code-group
```bash [pnpm]
pnpm add vue-router
```

```bash [npm]
npm install vue-router
```
::

### 添加 Vite 插件

在 `vite.config.ts` 注册 `movk()` 插件：

::code-group
```ts [vite.config.ts (Vite)]
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import movk from '@movk/nuxt/vite'

export default defineConfig({
  plugins: [
    vue(),
    movk()
  ]
})
```

```ts [vite.config.ts (Inertia)]
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import movk from '@movk/nuxt/vite'

export default defineConfig({
  plugins: [
    vue(),
    movk({
      router: 'inertia'
    })
  ]
})
```
::

::tip
`movk()` 内部注册了 `unplugin-auto-import` 与 `unplugin-vue-components`，会生成 `auto-imports.d.ts` 与 `components.d.ts`。建议将它们加入 `.gitignore` 并在 `tsconfig` 中 include。

```json [tsconfig.app.json]
{
  "include": ["src/**/*.ts", "src/**/*.vue", "auto-imports.d.ts", "components.d.ts"]
}
```

```bash [.gitignore]
auto-imports.d.ts
components.d.ts
```
::

::tip
若使用 TypeScript，可在 `tsconfig` 中为主题虚拟模块添加别名，以在 `vite.config.ts` 获得补全：

```json [tsconfig.node.json]
{
  "compilerOptions": {
    "paths": {
      "#build/movk-ui": ["./node_modules/.movk-ui/movk-ui"],
      "#build/ui": ["./node_modules/.nuxt-ui/ui"]
    }
  }
}
```
::

### 注册 Vue 插件

在入口注册 `@movk/nuxt/vue-plugin`（内部已链入 `@nuxt/ui/vue-plugin`）：

```ts [src/main.ts]
import './assets/css/main.css'
import { createApp } from 'vue'
import { createRouter, createWebHistory } from 'vue-router'
import movk from '@movk/nuxt/vue-plugin'
import App from './App.vue'

const router = createRouter({
  routes: [],
  history: createWebHistory()
})

createApp(App)
  .use(router)
  .use(movk)
  .mount('#app')
```

### 引入样式

`@movk/nuxt` 的样式入口已串联 Tailwind CSS、`@nuxt/ui` 与 movk 主题，只需一行导入：

```css [src/assets/css/main.css]
@import "@movk/nuxt";
```

### 用 `<UApp>` 包裹根组件

`<UApp>` 提供 Toast、Tooltip 与命令式 overlay 的全局上下文，`useMessageBox` 依赖它：

```vue [src/App.vue]
<template>
  <UApp>
    <RouterView />
  </UApp>
</template>
```

::note
根容器建议加 `class="isolate"`（`index.html` 的 `#app`），确保样式作用域与 overlay 层叠正确。
::

## 与 Nuxt 模式的差异

| 能力 | Nuxt 模式 | Vue / Vite 模式 |
| --- | --- | --- |
| 安装 / 注册 | `modules: ['@movk/nuxt']` | `vite.config` 的 `movk()` + `app.use(movkVuePlugin)` |
| 配置入口 | `nuxt.config` 的 `movk: {}` | `movk()` 选项 |
| 独立 UI 组件（AutoForm、DataTable、SearchForm、DatePicker、ColorChooser、MessageBox、PillGroup、Popconfirm、StarRating、SlideVerify、6 个 input 增强、ThemePicker） | ✅ | ✅ 自动导入 |
| Composables 自动导入 | ✅ | ✅（`useAutoForm` / `useTheme` / `useDateFormatter` / `useMessageBox`） |
| **API 集成域**：`useApiFetch` / `useLazyApiFetch` / `useClientApiFetch`、`useUploadWithProgress` / `useDownloadWithProgress`、`$api` 插件、鉴权、错误 toast、端点与拦截器 | ✅ | ❌ 不支持 |
| 主题 `useTheme` / ThemePicker / `tv()` | ✅，含 SSR 防闪脚本 | ✅（客户端恢复；无 SSR 防闪；localStorage key 前缀取 `site.name`，默认 `movk`） |
| CSS 注入 | 模块自动注入 | 手动 `@import "@movk/nuxt"` |
| 链接 / 路由 | Nuxt 路由 | `router` 选项：`true`（vue-router）/ `false`（`<a>`）/ `'inertia'` |

::note
关闭 `autoImport` 后仍可显式导入：`import { useTheme } from '@movk/nuxt/composables/useTheme'`。
::

## 选项

在 `vite.config.ts` 的 `movk()` 中传入选项进行配置。

### `prefix` 组件前缀

组件名前缀。

- 默认：`M`

```ts [vite.config.ts]
movk({
  prefix: 'M'
})
```

### `site` 站点信息

站点名称，影响主题持久化的 localStorage key 前缀（`<name>-ui-*`）。

```ts [vite.config.ts]
movk({
  site: { name: 'my-app' }
})
```

### `colorMode` 暗色模式

启用或禁用基于 `@vueuse/core` 的暗色模式集成。

- 默认：`true`

```ts [vite.config.ts]
movk({
  colorMode: false
})
```

### `router` 路由集成

配置链接组件的路由行为，适用于不使用 `vue-router` 的场景（如 Inertia.js）。

- 默认：`true`

| 值 | 说明 |
| --- | --- |
| `true` | 使用 `vue-router` 导航与 `RouterLink`（需安装 `vue-router`）。 |
| `false` | 禁用路由集成，链接渲染为 `<a>` 标签。 |
| `'inertia'` | 使用 Inertia.js 的 `Link` 组件导航。 |

```ts [vite.config.ts]
movk({
  router: false
})
```

::note
设为 `false` 或 `'inertia'` 时，无需安装 `vue-router`。
::

### `theme` 主题

`theme.colors`（颜色别名）、`theme.defaultVariants`（默认变体）、`theme.prefix`（与 Tailwind 前缀一致）等，对齐 Nuxt 模式 `movk.theme`。

```ts [vite.config.ts]
movk({
  theme: {
    defaultVariants: {
      color: 'neutral',
      size: 'sm'
    }
  }
})
```

### `autoImport` Composable 自动导入

禁用 Composable 自动导入，或自定义 `unplugin-auto-import` 选项。

- 默认：`{}`

```ts [vite.config.ts]
movk({
  autoImport: false
})
```

::note
禁用后仍可显式从 `@movk/nuxt/composables` 导入。
::

### `components` 组件自动导入

禁用组件自动导入，或自定义 `unplugin-vue-components` 选项。

- 默认：`{}`

```ts [vite.config.ts]
movk({
  components: false
})
```

::note
禁用后仍可显式导入，如 `import MAutoForm from '@movk/nuxt/components/AutoForm.vue'`。
::
