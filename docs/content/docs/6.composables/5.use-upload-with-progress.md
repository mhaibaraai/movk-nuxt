## Usage

`useUploadWithProgress` 基于原生 `XMLHttpRequest` 实现，复用 `@movk/nuxt` 的端点、认证、业务校验、Toast、`movk:api:*` hook 体系。返回的 `data` 是按 `dataKey` 解包后的业务数据，与 `useApiFetch<T>` 一致。

```vue
<script setup lang="ts">
const { progress, status, data, error, upload, abort } = useUploadWithProgress<{
  files: Array<{ name: string, size: number }>
}>()

async function handleUpload(files: File[]) {
  const { data, error: err, aborted } = await upload('/api/upload', files, {
    fieldName: 'files',
    onSuccess: (d) => console.log('上传成功:', d.files)
  })

  if (aborted) return
  if (err) console.error(err)
  else console.log(data?.files)
}
</script>

<template>
  <input type="file" multiple @change="e => handleUpload(Array.from(e.target.files))">
  <UProgress v-if="status === 'pending'" :model-value="progress" :max="100" />
  <UButton v-if="status === 'pending'" @click="abort">取消</UButton>
</template>
```

::note
适用于需要展示上传进度的场景。对于无需进度的小文件，可以直接使用 [`$api`](/docs/api/api-plugin) 发起请求。
::

## 单文件 vs 多文件

```ts
// 单文件
await upload('/api/upload', file, { fieldName: 'file' })
// FormData: file=<File>

// 多文件（同字段名追加）
await upload('/api/upload', [f1, f2, f3], { fieldName: 'files' })
// FormData: files=<File>, files=<File>, files=<File>
```

## 业务校验

成功响应（默认 `code` 在 `[200, 0]`）按 `dataKey` 解包，触发 `onSuccess` 与 `movk:api:response` hook；失败响应抛 `ApiError`，触发 `onError` 与 `movk:api:error` hook。

`skipBusinessCheck: true` 跳过业务码校验，`data` 取整个响应体。

业务码规则由[模块配置](/docs/getting-started/configuration#response)的 `response` 选项控制。

## 取消上传

```ts
const { status, upload, abort } = useUploadWithProgress()

upload('/api/upload', largeFile)

function handleCancel() {
  if (status.value === 'pending') abort()
}
```

取消后：

- `status` 变为 `'aborted'`，`progress` 重置为 `0`
- `error` 保持 `null`（不视为失败）
- 不触发 `onSuccess` / `onError`，不弹 Toast
- `upload()` 返回 `{ data: null, error: null, aborted: true }`

## 认证

自动从 session 读取 token 注入到 `Authorization` 头；可在 `headers` 中覆盖同名头（用户优先）。

```ts
await upload('/api/upload', file, {
  headers: { Authorization: 'Bearer override' }
})
```

## API

### useUploadWithProgress&lt;T&gt;()

`useUploadWithProgress<T>(): { progress, status, data, error, upload, abort }`{lang="ts-type"}

#### Returns

::field-group
  ::field{name="progress" type="Ref<number | null>"}
  上传进度，0-100；类型与下载侧对齐保留 `null` 形态，上传场景实际不会出现。
  ::

  ::field{name="status" type="Ref<TransferStatus>"}
  传输状态：`'idle' | 'pending' | 'success' | 'error' | 'aborted'`。
  ::

  ::field{name="data" type="Ref<T | null>"}
  按 `dataKey` 解包后的业务数据。
  ::

  ::field{name="error" type="Ref<ApiError | Error | null>"}
  错误对象；业务错误为 `ApiError`，HTTP/网络错误为 `Error`。中止时为 `null`。
  ::

  ::field{name="upload()" type="(url: string, files: File | File[], options?: UploadWithProgressOptions<T>) => Promise<TransferResult<T>>"}
  执行上传。
    ::collapsible
      **Parameters**

      ::field-group
        ::field{name="url" type="string" required}
        ::

        ::field{name="files" type="File | File[]" required}
        ::

        ::field{name="options" type="UploadWithProgressOptions<T>"}
          ::collapsible
            ::field-group
              ::field{name="fieldName" type="string"}默认 `'file'`。::
              ::field{name="fields" type="Record<string, string | Blob>"}::
              ::field{name="headers" type="Record<string, string>"}用户优先于自动 auth 头。::
              ::field{name="toast" type="RequestToastOptions | false"}::
              ::field{name="endpoint" type="string"}::
              ::field{name="skipBusinessCheck" type="boolean"}::
              ::field{name="timeoutMs" type="number"}超时毫秒数，0 或省略不超时。::
              ::field{name="withCredentials" type="boolean"}::
              ::field{name="onSuccess" type="(data: T) => void"}解包后的业务数据。::
              ::field{name="onError" type="(error: ApiError | Error) => void"}::
            ::
          ::
        ::
      ::

      **Returns**

      `Promise<TransferResult<T>>`：`{ data: T | null, error: ApiError | Error | null, aborted: boolean }`。
    ::
  ::

  ::field{name="abort()" type="() => void"}
  中止当前上传，`status` 切到 `'aborted'`。
  ::
::
